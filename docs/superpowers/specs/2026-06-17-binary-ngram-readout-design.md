# Binary n-gram Tokens + FF Bit-Readout — Design

Date: 2026-06-17
Location: `forward-forward/index.html` (reworks the tokenization, training signal, and prediction of the Forward-Forward char-LM)
Related: [2026-06-17-forward-forward-char-lm-design.md](2026-06-17-forward-forward-char-lm-design.md), [2026-06-17-muon-optimizer-design.md](2026-06-17-muon-optimizer-design.md)

## Goal

Replace the one-hot character vocabulary with a **dense binary** representation so the model
can train on **bytes (n=1), bigrams (n=2), or trigrams (n=3)**, selectable in the UI. A token
of `n` bytes is `8n` bits. The network emits the next token directly on **8n sigmoid output
neurons** (threshold 0.5 → bit), via a shallow readout head on top of Forward-Forward-trained
hidden layers — no one-hot, no 256ⁿ candidate enumeration.

## Why bits

A full one-hot over 256ⁿ is infeasible (65k–16M-wide vectors; a forward pass per candidate at
prediction). Binary encoding keeps input and output tiny (trigram, K=8 → 192 input bits, 24
output bits) and removes candidate search: prediction is a single forward pass.

## Tokenization

- **Token unit** `n ∈ {1,2,3}` chosen in the UI (Bytes / Bigrams / Trigrams).
- The corpus text is encoded to **UTF-8 bytes**. The byte stream is split into
  **non-overlapping** `n`-byte tokens; any trailing remainder (`len % n` bytes) is **dropped**.
- Each token → **8n bits**, byte-by-byte, **MSB-first** (bit `b` of byte `v` = `(v >> (7-b)) & 1`),
  values `0/1`. Decoding reverses this.
- There is **no vocabulary**: bits come directly from bytes. The one-hot path
  (`buildVocab`, `charToIndex`, `inputDim`, `encode`, `FFNet.predictDistribution`,
  `FFNet.sample`, `vocabCap`, UNK) is **removed**.

## FF hidden layers (reused core, new input + signal)

- **Input** to the FF stack = the previous **K tokens** concatenated as bits → `K·8n` input
  bits. The next token is the *output*, so it is **not** part of the input.
- Hidden layers are unchanged `FFLayer`s trained by **Forward-Forward goodness**
  (`g = Σ ReLU(Wx+b)²`, local per-layer logistic loss, L2-normalized activations between
  layers) using the **user-selected optimizer** (Muon / Adam / SGD — the existing selector
  governs these layers).
- **Positive** sample = a real contiguous `K`-token context window from the stream.
  **Negative** sample = a context of `K` tokens, each drawn from a **random position** in the
  token stream (an incoherent context). Hidden units thus learn to encode genuine sequential
  structure.

## Bit-readout head (new output)

- A single linear layer: weight `Wr` of shape `8n × R` and bias `br` of length `8n`, where
  `R` = the readout feature dimension. **Readout input** = the **concatenated L2-normalized
  activations of all hidden layers** (so `R = Σ layer widths`).
- Output = `sigmoid(Wr · features + br)` → `8n` probabilities; **bit = (p > 0.5)**.
- Trained by **binary cross-entropy** against the true next token's bits. This is a **shallow,
  local head**: its gradient updates only `Wr`/`br`; it does **not** backpropagate into the FF
  hidden layers (the hidden features are treated as a fixed extractor for the readout step).
  This matches Hinton's FF recipe (a trained linear readout on FF features).
- Readout optimizer: **Adam** (bias-corrected, β1=0.9, β2=0.999, ε=1e-8). The readout is small;
  the Muon/Adam/SGD selector applies to the FF *hidden* layers, not the readout.
- BCE gradient (per training example, sigmoid output `p`, target bit `y`): `dL/dlogit = p − y`;
  `dWr[i][j] += (p_i − y_i)·feat_j`, `dbr[i] += (p_i − y_i)`. Averaged over the batch, then one
  Adam step.

## Prediction / generation

- Context → FF `forwardAll` → readout → `8n` bit-probabilities.
  - **Greedy:** bit = `p > 0.5`.
  - **Temperature** (UI slider): per-bit `Bernoulli(sigmoid(logit / temp))` sampling for
    variety (low temp → greedy/argmax-ish; high temp → more random). `logit = ln(p/(1−p))`.
- Decode the `8n` sampled bits → `n` bytes → append to the output byte buffer; slide the
  context by one token and repeat. One forward pass per generated token.
- Generated bytes are decoded to text with a UTF-8 decoder that tolerates invalid sequences
  (replacement char) for display.

## UI

- New **Token unit** `<select id="tokenUnit">`: Bytes (8-bit, default), Bigrams (16-bit),
  Trigrams (24-bit). Changing it rebuilds the model (like changing layer count).
- The **next-char distribution** panel becomes a **per-bit probability** view: `8n` bars, one
  per output bit, labeled by byte/bit position; height = `p`.
- Generation emits `n` bytes per step; the seed prompt is tokenized the same way (its trailing
  partial token is left-padded/handled like other contexts).
- Optimizer selector, layer/width/K/LR/batch controls, Start/Pause/Reset, charts, save/load
  remain.

## Stats

- Keep FF **goodness separation** (mean positive vs negative goodness per layer) for the hidden
  layers.
- Replace "top-1 accuracy" with two readout metrics measured on held-out windows:
  - **bit-accuracy** — fraction of the `8n` output bits predicted correctly (averaged).
  - **exact-token accuracy** — fraction of windows where **all** `8n` bits are correct.

## Persistence

- `serializeNet`/`deserializeNet` extend to include the **readout** (`Wr`, `br`) and the
  **token unit** `n`. `weightsCompatible` checks that the saved `n` and `inDim` match the
  current model before loading. (Optimizer choice remains a runtime knob, not serialized.)

## Components

All in `forward-forward/index.html`; engine in `<script id="shared-code">`, UI in the second
`<script>`.

- **Tokenizer (shared-code):**
  - `textToBytes(text) → Uint8Array` and `bytesToText(bytes) → string` (UTF-8, tolerant decode).
  - `tokenizeBytes(bytes, n) → number[][]` — array of `n`-byte tokens (remainder dropped).
  - `tokenToBits(token, n) → Float64Array` (length `8n`, MSB-first) and
    `bitsToToken(bits, n) → number[]` (threshold 0.5 → bytes).
  - `encodeContextBits(tokenBitsArray, K, bitsPerTok) → Float64Array` — concat of `K` token
    bit-vectors, left-padded with zero tokens when short.
- **Readout (shared-code):** `class Readout` with `Wr`, `br`, Adam state; `predict(features) →
  Float64Array` (sigmoid probs); `accumulate(features, targetBits)`; `step(lr)`; `zeroGrad()`.
- **FFNet:** add `readoutFeatures(acts) → Float64Array` (concat of L2-normalized layer
  activations). Keep `forwardAll`. Remove `predictDistribution`/`sample`.
- **Trainer:** builds the token stream + `FFNet` + `Readout` from `cfg.text` and `cfg.n`.
  `stepBatch` trains hidden layers (positive real window / negative random-position context)
  AND the readout (features of the real context → true next-token bits). Metrics:
  `evalBitAccuracy(count)`, `evalTokenAccuracy(count)`. `generate` uses the readout.
- **UI:** token-unit selector + per-bit probability panel; wire `n` into `Ctrl.build()`.

## Testing (`test.mjs`)

- `tokenToBits`/`bitsToToken` round-trip for n=1,2,3 (random bytes via seeded RNG).
- `tokenizeBytes` non-overlapping split + remainder dropped; `textToBytes`/`bytesToText`
  round-trip on ASCII and a multi-byte char.
- `encodeContextBits` length = `K·8n`, left-pad behavior.
- **Readout overfits**: on a trivial repeating byte pattern, after training, exact-token
  accuracy on the pattern rises well above chance (n=1).
- FF goodness still separates real vs random-position contexts (mean pos > mean neg after
  training).
- The optimizer selector still drives the hidden layers (the existing "optimizer arg honored"
  and Newton-Schulz tests remain green); Muon/Adam/SGD all run under the new Trainer.
- save/load round-trip preserves readout + token unit; `weightsCompatible` rejects a different
  `n`.

## Non-goals

- No one-hot / vocabulary mode retained (binary is the only mode).
- No backprop through the FF hidden stack (readout is a shallow local head).
- No GPU/WebGL; no external libraries; still works from `file://`.
- Token unit beyond trigrams (n>3) is out of scope.
